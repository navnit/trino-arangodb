# M2: Filter + Limit + Projection Pushdown Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Push `WHERE`/`LIMIT`/nested-field projection from Trino into AQL wherever it is safe to do so with identical SQL semantics, per spec §6.1–§6.3, while leaving anything unsafe (nullable-restricted domains, string ranges, DECIMAL/ARRAY/ROW predicates) in Trino's residual filter.

**Architecture:** `ArangoTableHandle` gains `constraint`/`limit` fields; `ArangoColumnHandle` gains a `path` field (dotted AQL document path, distinct from its Trino-facing `name`). `ArangoMetadata.applyFilter/applyLimit/applyProjection` classify what can be pushed and store it on the handle; `AqlBuilder.buildScan` renders `FILTER`/`LIMIT`/`RETURN {...}` from that handle plus the exact columns Trino requests. This last part is a real architectural break from M1: M1 always ran `RETURN d` (the whole document) and let `ArangoPageSource` pick fields out of a `BaseDocument` afterward; M2 always renders an explicit `RETURN {...}` object literal built from the requested columns (needed for nested-field projection), so the cursor type changes from `ArangoCursor<BaseDocument>` to `ArangoCursor<Map>` end-to-end.

**Tech Stack:** Java 24, Trino SPI 476 (`ConnectorMetadata.applyFilter/applyLimit/applyProjection`, `TupleDomain`/`Domain`/`ValueSet`/`Range`, `ConnectorExpression`/`Variable`/`FieldDereference`), ArangoDB Java driver 7.x (AQL bind vars), JUnit 5 + AssertJ + Testcontainers (no mocking framework).

## Global Constraints

- Every `apply*` hook may only claim work it can reproduce with **identical SQL semantics**; when in doubt, return `Optional.empty()` (decline — Trino keeps doing the work itself). (spec §2.3)
- AQL's cross-type total ordering is `null < bool < number < string < array < object`. Any numeric range predicate pushed into AQL must be guarded with `IS_NUMBER(...)` so a non-numeric value stored under the same field can never silently satisfy a numeric `>`/`<`. (spec §6.1)
- String range predicates (`<`, `>`, `BETWEEN`, but not `=`/`IN`) are **never** pushed in M2 — AQL's default string comparison and Trino's VARCHAR codepoint comparison are not guaranteed to agree, and reconciling that is out of scope for this milestone. Equality/IN is safe and unguarded for every type, since `==`/`IN` never coerce.
- DECIMAL, ARRAY, and ROW-typed predicates are never pushed. `ArangoPageSourceProvider.checkMaterializable` already refuses to materialize ARRAY/ROW/DECIMAL *values*, and DECIMAL bind-var marshaling through the ArangoDB driver is untested; pushing a filter that can never be verified against a real read is out of scope.
- `ArangoSplitManager` always emits exactly one `ArangoSplit` per table (unchanged in M2), so any AQL `LIMIT` we push is always exactly satisfied — `applyLimit`'s `limitGuaranteed` is always `true`.
- No mocking framework anywhere in the test suite. Tests that need to avoid a live ArangoDB server use hand-written `ArangoClient`/`SchemaResolver` subclasses as test doubles (see existing `ArangoMetadataTest`); true end-to-end tests use `TestingArangoServer` + `DistributedQueryRunner` via Testcontainers (Docker must be running).
- Do not touch the `pom.xml` dependency-mediation pins or their comments (jackson-annotations/opentelemetry pins, io.airlift `336` pins, junit pins) — out of scope for M2.
- `arangodb.string-range-pushdown` (spec §8, opt-in string range pushdown) is **not implemented** in M2: it is off by default and not required by the exit criteria below, so implementing it now would add a config property, a new `ArangoMetadata` constructor argument, and cascading test-signature churn for a feature nothing in this milestone exercises. Deferred to a later milestone.
- IS NULL / IS NOT NULL predicates are **never** pushed in M2, regardless of column type. `ArangoPageSource.appendValue`'s lenient type-mismatch coercion means a raw stored value's null-ness in ArangoDB (what AQL's `== null`/`!= null` would test) and Trino's post-coercion null-ness (what the SQL predicate actually means) can disagree for any value outside `SchemaResolver`'s sample, or any value the inferred type doesn't fit — silently dropping or including rows with no error surfaced. Unlike ranges, there is no cheap guard that closes this gap (a raw-value type guard like `IS_NUMBER` doesn't mirror `appendValue`'s coercion exactly: e.g. a BIGINT column with an out-of-sample float value passes `IS_NUMBER` but still coerces to NULL in Trino). Left residual, Trino re-applies these predicates itself post-coercion, exactly matching M1's behavior. (Found and resolved during Task 3's review — see `.superpowers/sdd/progress.md`.)
- Exit criteria (spec §10): `isFullyPushedDown()` passes for guarded predicates (equality, numeric range); IS NULL/IS NOT NULL remain correct via Trino's residual filter (not pushed — see above); mixed-type filter scenarios (a column that types as numeric from its sample but has a non-numeric value at query time) still return SQL-correct rows via the `IS_NUMBER` guard.

### Task 2: Migrate the read path from `RETURN d`/`BaseDocument` to `RETURN {...}`/`Map`

Everything M2 pushes (filter, limit, projection) is rendered by `AqlBuilder.buildScan`. Nested-field projection (Task 5) requires the AQL query to explicitly name which document paths to return — `RETURN d` (the whole document) can't express "give me `address.city` but not the rest of `address`". So this task switches `AqlBuilder` to always build an explicit `RETURN {...}` object literal from the requested columns, for *every* scan, not just projected ones. That, in turn, means the ArangoDB driver returns each row as a `Map` instead of a `BaseDocument`, since a custom `RETURN {...}` shape is no longer a real document — so `ArangoClient.query` and `ArangoPageSource` change their cursor's element type from `BaseDocument` to `Map`. This is the highest-blast-radius change in the whole plan (every read goes through it) and carries zero pushdown logic of its own — landing it alone, and confirming the *existing* M1 integration tests (`ArangoConnectorQueryTest`) still pass unmodified, isolates any regression here from the pushdown logic added in later tasks.

**Files:**
- Modify: `src/main/java/io/arango/trino/aql/AqlBuilder.java`
- Modify: `src/main/java/io/arango/trino/client/ArangoClient.java:63-65` (one method)
- Modify: `src/main/java/io/arango/trino/ArangoPageSource.java`
- Modify: `src/test/java/io/arango/trino/aql/AqlBuilderTest.java` (rewrite the one existing test to match the new output shape, add one new test)

**Interfaces:**
- Consumes: `ArangoTableHandle.constraint()`/`limit()` (Task 1) — untouched by this task (no `FILTER`/`LIMIT` rendering yet).
- Produces: `AqlBuilder.buildScan(ArangoTableHandle, List<ArangoColumnHandle>)` now renders `RETURN {"<name>": d[<path segments>], ...}` (or `RETURN {}` for an empty column list) instead of `RETURN d`.
- Produces: `ArangoClient.query(String database, String aql, Map<String, Object> bindVars)` now returns `ArangoCursor<Map>` (was `ArangoCursor<BaseDocument>`).
- Produces: `new ArangoPageSource(ArangoCursor<Map> cursor, List<ArangoColumnHandle> columns)` (constructor signature changed; `_key`/`_id`/`_rev` are no longer special-cased since every column, including system attributes, is now uniformly aliased in the `RETURN {...}` clause and read back via `row.get(column.name())`).

- [ ] **Step 1: Rewrite `AqlBuilder.java`'s `buildScan` to build the `RETURN {...}` clause**

```java
package io.arango.trino.aql;

import io.arango.trino.handle.ArangoColumnHandle;
import io.arango.trino.handle.ArangoTableHandle;

import java.util.List;
import java.util.Map;

public class AqlBuilder {
    public record AqlQuery(String aql, Map<String, Object> bindVars) {}

    public AqlQuery buildScan(ArangoTableHandle table, List<ArangoColumnHandle> columns) {
        Map<String, Object> bindVars = new java.util.LinkedHashMap<>();
        bindVars.put("@col", table.table());

        StringBuilder aql = new StringBuilder("FOR d IN @@col");
        aql.append(" RETURN ").append(buildReturnClause(columns));
        return new AqlQuery(aql.toString(), bindVars);
    }

    private static String buildReturnClause(List<ArangoColumnHandle> columns) {
        if (columns.isEmpty()) {
            return "{}";
        }
        StringBuilder sb = new StringBuilder("{");
        for (int i = 0; i < columns.size(); i++) {
            if (i > 0) {
                sb.append(", ");
            }
            ArangoColumnHandle column = columns.get(i);
            sb.append(quoteAqlString(column.name())).append(": ").append(documentAccessor(column.path()));
        }
        return sb.append("}").toString();
    }

    // `path` is treated as one literal document attribute name via AQL bracket notation, which
    // does NOT re-split on dots (only AQL dot notation like d.a.b would). Through Task 4, path
    // always equals a column's flat name — there is no multi-segment path yet. Task 5 introduces
    // real nested paths and must represent them as pre-split segments (not a dot-joined string)
    // to avoid ambiguity between a path separator and a literal "." in a field name — see the
    // Global Constraints note on this. (Fixed post-Task-2 review; the original version here split
    // on "." unconditionally, silently mis-rendering any column whose literal field name contains
    // "." as a nested access.)
    private static String documentAccessor(String path) {
        return "d[" + quoteAqlString(path) + "]";
    }

    private static String quoteAqlString(String value) {
        return "\"" + value.replace("\\", "\\\\").replace("\"", "\\\"") + "\"";
    }
}
```

```java
package io.arango.trino;

import com.arangodb.ArangoCursor;
import io.arango.trino.handle.ArangoColumnHandle;
import io.trino.spi.Page;
import io.trino.spi.PageBuilder;
import io.trino.spi.block.BlockBuilder;
import io.trino.spi.connector.ConnectorPageSource;
import io.trino.spi.connector.SourcePage;
import io.trino.spi.type.*;

import java.util.List;
import java.util.Map;

import static io.airlift.slice.Slices.utf8Slice;
import static java.util.Objects.requireNonNull;

public class ArangoPageSource implements ConnectorPageSource {
    private static final int ROWS_PER_PAGE = 1024;

    private final List<ArangoColumnHandle> columns;
    private final List<Type> types;
    private final ArangoCursor<Map> cursor;
    private final PageBuilder pageBuilder;
    private boolean finished;
    private long completedBytes;

    public ArangoPageSource(ArangoCursor<Map> cursor, List<ArangoColumnHandle> columns) {
        this.cursor = requireNonNull(cursor, "cursor is null");
        this.columns = List.copyOf(columns);
        this.types = columns.stream().map(ArangoColumnHandle::type).toList();
        this.pageBuilder = new PageBuilder(types);
    }

    @Override
    public SourcePage getNextSourcePage() {
        int rows = 0;
        while (rows < ROWS_PER_PAGE && cursor.hasNext()) {
            @SuppressWarnings("unchecked")
            Map<String, Object> row = cursor.next();
            pageBuilder.declarePosition();
            for (int i = 0; i < columns.size(); i++) {
                ArangoColumnHandle col = columns.get(i);
                BlockBuilder out = pageBuilder.getBlockBuilder(i);
                appendValue(out, types.get(i), row.get(col.name()));
            }
            rows++;
        }
        if (!cursor.hasNext()) {
            finished = true;
        }
        if (rows == 0) {
            return null;
        }
        Page page = pageBuilder.build();
        completedBytes += page.getSizeInBytes();
        pageBuilder.reset();
        return SourcePage.create(page);
    }

    // lenient coercion: mismatched/absent value -> NULL
    private static void appendValue(BlockBuilder out, Type type, Object value) {
        if (value == null) {
            out.appendNull();
            return;
        }
        try {
            if (type.equals(BooleanType.BOOLEAN) && value instanceof Boolean b) {
                BooleanType.BOOLEAN.writeBoolean(out, b);
            }
            else if (type.equals(BigintType.BIGINT) && value instanceof Number n) {
                BigintType.BIGINT.writeLong(out, n.longValue());
            }
            else if (type.equals(DoubleType.DOUBLE) && value instanceof Number n) {
                DoubleType.DOUBLE.writeDouble(out, n.doubleValue());
            }
            else if (type instanceof VarcharType) {
                type.writeSlice(out, utf8Slice(String.valueOf(value)));
            }
            else {
                // Defensive fallback only: ArangoPageSourceProvider.checkMaterializable rejects
                // ARRAY/ROW/DECIMAL columns with a loud TrinoException before any query runs,
                // so a real ARRAY/ROW/DECIMAL value should never reach this branch. This
                // remains lenient-NULL for any other unanticipated type/value mismatch.
                out.appendNull();
            }
        }
        catch (RuntimeException e) {
            out.appendNull(); // lenient
        }
    }

    @Override public long getCompletedBytes() { return completedBytes; }
    @Override public long getReadTimeNanos() { return 0; }
    @Override public boolean isFinished() { return finished; }
    @Override public long getMemoryUsage() { return pageBuilder.getRetainedSizeInBytes(); }
    @Override public void close() { try { cursor.close(); } catch (Exception ignored) {} }
}
```

package io.arango.trino.aql;

import io.airlift.slice.Slice;
import io.arango.trino.handle.ArangoColumnHandle;
import io.arango.trino.handle.ArangoTableHandle;
import io.trino.spi.connector.ColumnHandle;
import io.trino.spi.predicate.Domain;
import io.trino.spi.predicate.Range;
import io.trino.spi.predicate.ValueSet;

import java.util.ArrayList;
import java.util.LinkedHashMap;
import java.util.List;
import java.util.Map;

public class AqlBuilder {
    public record AqlQuery(String aql, Map<String, Object> bindVars) {}

    public AqlQuery buildScan(ArangoTableHandle table, List<ArangoColumnHandle> columns) {
        Map<String, Object> bindVars = new LinkedHashMap<>();
        bindVars.put("@col", table.table());
        int[] counter = {0};

        StringBuilder aql = new StringBuilder("FOR d IN @@col");

        List<String> filters = new ArrayList<>();
        table.constraint().getDomains().ifPresent(domains -> {
            for (Map.Entry<ColumnHandle, Domain> entry : domains.entrySet()) {
                ArangoColumnHandle column = (ArangoColumnHandle) entry.getKey();
                filters.add("(" + renderDomain(column, entry.getValue(), bindVars, counter) + ")");
            }
        });
        if (!filters.isEmpty()) {
            aql.append(" FILTER ").append(String.join(" AND ", filters));
        }

        aql.append(" RETURN ").append(buildReturnClause(columns));
        return new AqlQuery(aql.toString(), bindVars);
    }

    // Only reachable for domain shapes ArangoMetadata.applyFilter already classified as
    // pushable, so this method trusts its input rather than re-verifying types: equality/IN
    // (any type) or a genuine range (BIGINT/DOUBLE only). IS NULL / IS NOT NULL are never
    // pushed (see ArangoMetadata.isPushable) and so never reach this method.
    private static String renderDomain(ArangoColumnHandle column, Domain domain, Map<String, Object> bindVars, int[] counter) {
        String accessor = documentAccessor(column.path());
        ValueSet values = domain.getValues();
        if (values.isDiscreteSet()) {
            List<Object> discrete = values.getDiscreteSet();
            if (discrete.size() == 1) {
                String v = bindValue(bindVars, counter, toBindValue(discrete.get(0)));
                return accessor + " == @" + v;
            }
            List<Object> converted = discrete.stream().map(AqlBuilder::toBindValue).toList();
            String v = bindValue(bindVars, counter, converted);
            return accessor + " IN @" + v;
        }
        List<String> rangeClauses = new ArrayList<>();
        for (Range range : values.getRanges().getOrderedRanges()) {
            rangeClauses.add(renderRange(accessor, range, bindVars, counter));
        }
        String combined = rangeClauses.size() == 1
                ? rangeClauses.get(0)
                : "(" + String.join(" OR ", rangeClauses) + ")";
        return "IS_NUMBER(" + accessor + ") AND " + combined;
    }

    private static String renderRange(String accessor, Range range, Map<String, Object> bindVars, int[] counter) {
        if (range.isSingleValue()) {
            String v = bindValue(bindVars, counter, toBindValue(range.getSingleValue()));
            return accessor + " == @" + v;
        }
        List<String> parts = new ArrayList<>();
        if (!range.isLowUnbounded()) {
            String v = bindValue(bindVars, counter, toBindValue(range.getLowBoundedValue()));
            parts.add(accessor + (range.isLowInclusive() ? " >= @" : " > @") + v);
        }
        if (!range.isHighUnbounded()) {
            String v = bindValue(bindVars, counter, toBindValue(range.getHighBoundedValue()));
            parts.add(accessor + (range.isHighInclusive() ? " <= @" : " < @") + v);
        }
        return parts.size() == 1 ? parts.get(0) : "(" + String.join(" AND ", parts) + ")";
    }

    // Trino represents VARCHAR predicate values as io.airlift.slice.Slice, not String; the
    // ArangoDB driver's bind-var marshaling has no notion of Slice, so it must be converted
    // before it can be handed to the driver. BIGINT/DOUBLE/BOOLEAN values are already the
    // right Java type (Long/Double/Boolean) and pass through unchanged.
    private static Object toBindValue(Object trinoValue) {
        return trinoValue instanceof Slice slice ? slice.toStringUtf8() : trinoValue;
    }

    private static String bindValue(Map<String, Object> bindVars, int[] counter, Object value) {
        String name = "v" + counter[0]++;
        bindVars.put(name, value);
        return name;
    }

    private static String buildReturnClause(List<ArangoColumnHandle> columns) {
        if (columns.isEmpty()) {
            return "{}";
        }
        StringBuilder sb = new StringBuilder("{");
        for (int i = 0; i < columns.size(); i++) {
            if (i > 0) {
                sb.append(", ");
            }
            ArangoColumnHandle column = columns.get(i);
            sb.append(quoteAqlString(column.name())).append(": ").append(documentAccessor(column.path()));
        }
        return sb.append("}").toString();
    }

    private static String documentAccessor(String path) {
        return "d[" + quoteAqlString(path) + "]";
    }

    private static String quoteAqlString(String value) {
        return "\"" + value.replace("\\", "\\\\").replace("\"", "\\\"") + "\"";
    }
}

```java
    @Override
    public Optional<ConstraintApplicationResult<ConnectorTableHandle>> applyFilter(
            ConnectorSession session, ConnectorTableHandle table, Constraint constraint) {
        ArangoTableHandle handle = (ArangoTableHandle) table;
        TupleDomain<ColumnHandle> newDomain = constraint.getSummary();
        if (newDomain.isNone() || newDomain.isAll()) {
            return Optional.empty();
        }

        Map<ColumnHandle, Domain> domains = newDomain.getDomains().orElseThrow();
        ImmutableMap.Builder<ColumnHandle, Domain> pushedBuilder = ImmutableMap.builder();
        ImmutableMap.Builder<ColumnHandle, Domain> residualBuilder = ImmutableMap.builder();
        for (Map.Entry<ColumnHandle, Domain> entry : domains.entrySet()) {
            ArangoColumnHandle column = (ArangoColumnHandle) entry.getKey();
            Domain domain = entry.getValue();
            if (isPushable(column.type(), domain)) {
                pushedBuilder.put(column, domain);
            }
            else {
                residualBuilder.put(column, domain);
            }
        }

        TupleDomain<ColumnHandle> pushed = TupleDomain.withColumnDomains(pushedBuilder.buildOrThrow());
        TupleDomain<ColumnHandle> residual = TupleDomain.withColumnDomains(residualBuilder.buildOrThrow());

        TupleDomain<ColumnHandle> newHandleConstraint = handle.constraint().intersect(pushed);
        if (newHandleConstraint.equals(handle.constraint())) {
            return Optional.empty(); // nothing new to push; avoid re-invoking applyFilter forever
        }

        ArangoTableHandle newHandle = handle.withConstraint(newHandleConstraint);
        return Optional.of(new ConstraintApplicationResult<>(newHandle, residual, constraint.getExpression(), false));
    }

    // Per spec §6.1: equality/IN is safe to push unguarded for every type (AQL's `==`/`IN`
    // are type-strict, never coerce). Numeric range needs an IS_NUMBER guard (AqlBuilder adds
    // it) because AQL's cross-type total order would otherwise let a non-numeric value stored
    // under the same field silently satisfy a numeric `>`/`<`. String range is never pushed
    // (ICU collation vs Trino codepoint-order mismatch, spec §6.1). DECIMAL/ARRAY/ROW are
    // never pushed -- materialization itself is unsupported (ArangoPageSourceProvider
    // .checkMaterializable) and DECIMAL bind-var marshaling is untested. A domain that allows
    // both null AND a restricted value set ("x = 5 OR x IS NULL") is not modeled in M2 and
    // stays in the residual.
    private static boolean isPushable(Type type, Domain domain) {
        // IS NULL / IS NOT NULL are never pushed: AQL's `== null` / `!= null` test the raw
        // stored value, while ArangoPageSource.appendValue leniently coerces any type-mismatched
        // value to Trino NULL. A value outside SchemaResolver's sample (or one that didn't fit the
        // inferred type) would answer these predicates differently in AQL than in Trino, silently
        // dropping or including rows. Left residual, Trino re-applies them post-coercion.
        if (domain.isAll()) {
            return false;
        }
        if (domain.isNullAllowed()) {
            return false;
        }
        if (domain.getValues().isAll()) {
            return false;
        }
        if (type instanceof VarcharType || type.equals(BooleanType.BOOLEAN)) {
            return domain.getValues().isDiscreteSet();
        }
        return type.equals(BigintType.BIGINT) || type.equals(DoubleType.DOUBLE);
    }
```

### Task 5: Nested projection pushdown — `ArangoMetadata.applyProjection`

Plain top-level column projection (`SELECT name, age`) is already handled "for free": `ArangoPageSourceProvider.createPageSource` already only requests exactly the columns Trino projects, and (since Task 2) `AqlBuilder` already only returns those columns' paths. So `applyProjection` in this connector only earns its keep for `FieldDereference` — nested fields like `SELECT address.city` — where without pushdown, Trino would need to materialize the *whole* `address` ROW itself and extract `.city` in a `ProjectNode` above the table scan. Since ROW materialization is unsupported (`checkMaterializable` throws `TrinoException(NOT_SUPPORTED)` for any *requested* ARRAY/ROW/DECIMAL column), that fallback path is a dead end — meaning a query that selects a nested field can only succeed at all if `applyProjection` successfully pushes the dereference down to a scalar leaf. This gives Task 6's integration test a correctness proof "for free": if `SELECT address.city ...` returns the right value, pushdown *must* have happened, since the non-pushed-down path would have thrown.

```java
    @Override
    public Optional<ProjectionApplicationResult<ConnectorTableHandle>> applyProjection(
            ConnectorSession session, ConnectorTableHandle table,
            List<ConnectorExpression> projections, Map<String, ColumnHandle> assignments) {
        ImmutableList.Builder<ConnectorExpression> newProjections = ImmutableList.builder();
        ImmutableList.Builder<Assignment> newAssignments = ImmutableList.builder();
        Map<String, ArangoColumnHandle> deduped = new LinkedHashMap<>();
        boolean progress = false;

        for (ConnectorExpression projection : projections) {
            Optional<ArangoColumnHandle> resolved;
            if (projection instanceof Variable variable) {
                resolved = assignments.get(variable.getName()) instanceof ArangoColumnHandle existing
                        ? Optional.of(existing) : Optional.empty();
            }
            else {
                resolved = resolveDereference(projection, assignments);
                progress |= resolved.isPresent();
            }
            if (resolved.isEmpty()) {
                return Optional.empty(); // an expression we can't represent -- decline the whole call
            }
            ArangoColumnHandle column = resolved.get();
            boolean isNewColumn = !deduped.containsKey(column.name());
            ArangoColumnHandle canonical = deduped.computeIfAbsent(column.name(), key -> column);
            if (isNewColumn) {
                newAssignments.add(new Assignment(canonical.name(), canonical, canonical.type()));
            }
            newProjections.add(new Variable(canonical.name(), canonical.type()));
        }

        if (!progress) {
            return Optional.empty(); // pure passthrough of existing columns: nothing new to report
        }
        return Optional.of(new ProjectionApplicationResult<>(table, newProjections.build(), newAssignments.build(), false));
    }

    // Walks a (possibly chained) FieldDereference down to its root Variable, resolving each
    // field index against the actual RowType field names so the result is a genuine multi-
    // segment AQL document path (e.g. List.of("address", "city")). Declines (Optional.empty())
    // whenever the chain doesn't root in a known column, indexes into a non-ROW type, hits an
    // anonymous row field, or bottoms out at a still-structured (ROW/ARRAY/DECIMAL) leaf --
    // those stay Trino-evaluated, consistent with checkMaterializable's existing ARRAY/ROW/
    // DECIMAL rejection. Segments are never joined into a String, so a field name that itself
    // contains a literal "." is not ambiguous (see AqlBuilder.documentAccessor).
    private static Optional<ArangoColumnHandle> resolveDereference(
            ConnectorExpression expression, Map<String, ColumnHandle> assignments) {
        List<Integer> fieldIndexes = new ArrayList<>();
        ConnectorExpression current = expression;
        while (current instanceof FieldDereference deref) {
            fieldIndexes.add(0, deref.getField());
            current = deref.getTarget();
        }
        if (!(current instanceof Variable base) || fieldIndexes.isEmpty()) {
            return Optional.empty();
        }
        if (!(assignments.get(base.getName()) instanceof ArangoColumnHandle baseColumn)) {
            return Optional.empty();
        }

        Type currentType = baseColumn.type();
        List<String> path = new ArrayList<>(baseColumn.path());
        StringBuilder name = new StringBuilder(baseColumn.name());
        for (int fieldIndex : fieldIndexes) {
            if (!(currentType instanceof RowType rowType)) {
                return Optional.empty();
            }
            RowType.Field field = rowType.getFields().get(fieldIndex);
            if (field.getName().isEmpty()) {
                return Optional.empty();
            }
            String fieldName = field.getName().get();
            path.add(fieldName);
            name.append('$').append(fieldName);
            currentType = field.getType();
        }
        if (currentType instanceof RowType || currentType instanceof ArrayType || currentType instanceof DecimalType) {
            return Optional.empty();
        }
        return Optional.of(new ArangoColumnHandle(name.toString(), currentType, false, path));
    }
```
